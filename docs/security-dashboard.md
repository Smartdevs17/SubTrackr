# Security Dashboard

The live dashboard is generated by `.github/workflows/security-scan.yml` on every
pull request, merge, scheduled run, and manual dispatch. The `Security Dashboard`
job uploads `security-dashboard.md` and writes the same table to the GitHub Actions
step summary.

## Dashboard Fields

| Field | Meaning |
| --- | --- |
| Tool | Scanner that produced the finding: Semgrep, OWASP ZAP, npm audit, cargo-audit, pip-audit, Snyk, or Trivy |
| Severity | Normalized finding severity used by the merge policy |
| Age | New in the current scan unless triaged into a follow-up issue |
| Target | File, package, image, or URL affected |
| Fix guidance | Scanner-provided remediation or the repo policy fallback |
| SLA | Required response window from `docs/SECURITY.md` |

## Current Gate Policy

| Severity | Count source | SLA | Gate |
| --- | --- | --- | --- |
| Critical | Aggregated scanner JSON reports | <24h | Blocks merge |
| High | Aggregated scanner JSON reports | <72h | Requires security-team review |
| Medium | Aggregated scanner JSON reports | Next scheduled release | Triage |
| Low | Aggregated scanner JSON reports | Best effort | Triage |
